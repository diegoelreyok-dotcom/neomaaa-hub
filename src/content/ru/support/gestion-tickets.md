## 1. Жизненный цикл тикета

### 1.1 Статусы тикета

Каждый тикет в Intercom проходит через следующие статусы:

```
НОВЫЙ → В РАБОТЕ → ОЖИДАНИЕ → ЭСКАЛИРОВАН → РЕШЕН → ЗАКРЫТ
 (1)      (2)        (3)        (4)          (5)      (6)
```

| Статус | Определение | Кто управляет | Максимальное время в этом статусе |
|--------|------------|---------------|-----------------------------------|
| **Новый** | Тикет только создан, не назначен или без ответа | Система / Агент | SLA первого ответа |
| **В работе** | Агент активно работает над тикетом | Назначенный агент | Согласно SLA решения |
| **Ожидание клиента** | Агент ответил, ожидает информацию от клиента | Агент | 48 часов до напоминания |
| **Ожидание внутреннее** | Агент ожидает ответ от L2 (Susana, Pepe, Finance) | Агент + L2 | 4 часа до повторного запроса |
| **Эскалирован** | Передан на более высокий уровень | L2/L3 | Согласно SLA эскалации |
| **Решен** | Проблема устранена, ожидается подтверждение клиента | Агент | 48 часов до закрытия |
| **Закрыт** | Клиент подтвердил решение или истек срок ожидания | Система / Агент | Окончательный |

### 1.2 Визуальная схема

```
Клиент отправляет сообщение
        |
        v
[НОВЫЙ] Тикет создается автоматически в Intercom
        |
        v
Автоматическая маршрутизация по языку (Правило 1/2 регламента)
        |
        v
Агент видит тикет в своей очереди
        |
        v
Агент открывает тикет → [В РАБОТЕ]
        |
        +--→ Может решить? --ДА--→ Решает → [РЕШЕН] → Подтверждение → [ЗАКРЫТ]
        |
        +--→ Нужна информация от клиента? → Запрашивает → [ОЖИДАНИЕ КЛИЕНТА]
        |                                                   |
        |                                   Клиент ответил? --ДА--→ [В РАБОТЕ]
        |                                                   |
        |                                          --НЕТ (48ч)--→ Напоминание → (еще 48ч) → [ЗАКРЫТ]
        |
        +--→ Нужна внутренняя информация? → Эскалирует/консультирует → [ОЖИДАНИЕ ВНУТРЕННЕЕ]
        |                                                   |
        |                                    L2 ответил? --ДА--→ [В РАБОТЕ]
        |                                                   |
        |                                          --НЕТ (4ч)--→ Повтор в Slack → (еще 4ч) → Principals
        |
        +--→ За пределами компетенции? → [ЭСКАЛИРОВАН]
                                      |
                                      v
                              L2 берет контроль → Решает → [РЕШЕН] → [ЗАКРЫТ]
```

## 2. Создание и классификация

### 2.1 Как создается тикет

| Источник | Создание | Автоматический тег |
|----------|----------|-------------------|
| Онлайн-чат (Intercom Messenger) | Автоматическое | Нет (агент должен проставить теги) |
| Email на [email protected] | Автоматическое (Intercom захватывает) | `email` |
| Контактная форма на сайте | Автоматическое | `web-form` |
| Агент создает вручную (из WhatsApp или звонка) | Ручное | Агент должен проставить теги |

### 2.2 Немедленная классификация

В первые 2 минуты после открытия тикета агент должен:

1. **Прочитать сообщение полностью** -- не отвечать без полного понимания
2. **Проставить обязательные теги:**
   - Тег типа: `kyc`, `deposit`, `withdrawal`, `mt5-platform` и т.д.
   - Тег приоритета: `priority:urgent`, `priority:high`, `priority:medium`, `priority:low`
   - Тег языка: `lang:es`, `lang:en`
3. **Проверить VIP-статус:** Проверить атрибут `vip` в профиле контакта
4. **Проверить историю:** Найти предыдущие тикеты того же клиента

### 2.3 Критерии приоритета (краткая справка)

| Приоритет | Экспресс-критерий | Пример |
|-----------|-------------------|--------|
| **Срочный** | Деньги под угрозой ПРЯМО СЕЙЧАС | Незачисленный депозит >24ч, невозможность закрыть позицию |
| **Высокий** | Невозможность торговать | KYC отклонен, MT5 не подключается, вывод задержан >48ч |
| **Средний** | Может торговать, но с затруднениями | Вопрос по способу оплаты, изменение кредитного плеча |
| **Низкий** | Информационный | Часы торговли, типы счетов, предварительная регистрация |

## 3. Назначение

### 3.1 Правила автоматического назначения

| Условие | Назначение |
|---------|-----------|
| Испанский язык | Агент 1 (LATAM) |
| Английский язык | Агент 2 (Европа/Ближний Восток) |
| VIP-клиент Gold/Platinum | Назначенный менеджер (Rocio/Marilyn) |
| Нерабочее время | Без назначения → первый агент следующей смены |

### 3.2 Ручное переназначение

Тикет переназначается, когда:
- Первоначальный агент не может его решить и требуется другой агент с профильными знаниями
- Передача смены (см. протокол передачи в регламенте)
- Клиент просит другого собеседника (всегда уважать)

**Процедура переназначения:**
1. Добавить внутреннюю заметку с резюме дела
2. Изменить назначенного агента в Intercom
3. Уведомить нового агента через Slack
4. Сообщить клиенту: «Я переведу вас на [имя], специалиста по [направление]»

### 3.3 Неназначенные тикеты

Неназначенные тикеты -- ответственность первого агента, начинающего смену.

**Протокол начала смены:**
1. Проверить очередь неназначенных тикетов
2. Отсортировать по давности (самый старый первым)
3. Отфильтровать по приоритету (срочные в первую очередь)
4. Назначить себе тикеты по своему языку/региону
5. Ответить на срочные в первые 10 минут смены

## 4. Активное управление тикетом

### 4.1 Первый ответ

Первый ответ -- самый важный. Он формирует восприятие клиентом качества сервиса.

**Чек-лист первого ответа:**
- [ ] Приветствие с именем клиента
- [ ] Подтверждение понимания запроса (перефразировать при сложном запросе)
- [ ] Дать ответ или сообщить, что ведется расследование
- [ ] Если ведется расследование, указать ориентировочное время обновления
- [ ] Спросить, есть ли дополнительные вопросы

### 4.2 Обновления в процессе работы

Если тикет не решается при первом взаимодействии:

| Ситуация | Действие | Периодичность |
|----------|----------|---------------|
| Ведется внутреннее расследование | Сообщить клиенту, что работа ведется | Каждые 2 часа |
| Ожидание ответа от L2 | Сообщить клиенту и указать сроки | При эскалации + каждые 4 часа |
| Процесс занимает более 1 дня | Отправить обновление даже при отсутствии новостей | Каждые 12 часов |
| VIP-тикет | Более частые обновления | Каждые 2 часа минимум |

**Шаблон обновления:**
```
Здравствуйте, [имя], хочу предоставить вам обновление по вашему делу.

Мы [описание текущей работы]. Окончательного решения пока нет, 
но подтверждаю, что ваш вопрос активно обрабатывается.

Сообщу, как только появятся новости. Ориентировочный срок: [время].
```

### 4.3 Внутренние заметки

Каждое существенное действие по тикету должно сопровождаться внутренней заметкой в Intercom.

**Что документировать во внутренних заметках:**
- Изложение того, что сообщает клиент
- Предпринятые действия (что проверил, где искал, с кем консультировался)
- Результат проверки
- Если эскалировано: кому, когда, почему
- Если получен ответ от L2: что сказано

**Формат внутренней заметки:**
```
[ДАТА] [АГЕНТ]
Действие: [что сделал]
Результат: [что обнаружил]
Далее: [что осталось / что ожидаю]
```

## 5. Эскалация

### 5.1 Когда эскалировать

Эскалировать, когда:
- Невозможно решить за 15 минут инструментами L1
- Требуется доступ или авторизация, которых нет
- Вопрос касается комплаенса, регулирования или юридических аспектов
- Клиент попросил соединить с руководителем
- Это формальная жалоба (см. протокол в `manejo-quejas.md`)

### 5.2 Процедура эскалации

1. **Задокументировать во внутренней заметке:**
   - Что пытался решить
   - Какой информацией располагаю
   - Почему эскалирую
   - Что требуется от L2

2. **Сообщить L2 через Slack:**
   ```
   @[имя] -- Эскалированный тикет: [ссылка на тикет]
   Клиент: [имя] -- Счет: [MT5#]
   Проблема: [резюме в 1 строку]
   Что пытался: [резюме]
   Что требуется: [действие L2]
   Приоритет: [срочный/высокий/средний]
   ```

3. **Сообщить клиенту:**
   ```
   [Имя], я передал ваш вопрос в отдел [направление], 
   располагающий необходимыми инструментами для решения.
   
   Я лично отслеживаю ваш вопрос и свяжусь с вами 
   в течение [время]. Номер вашего дела по-прежнему #[номер].
   ```

4. **Добавить теги:** `escalated` + `waiting-internal`

5. **Сопровождение:** Если L2 не ответил в течение 2 часов, повторный запрос через Slack. Если нет ответа через 4 часа, эскалация Principals через Susana.

### 5.3 Матрица эскалации (краткая справка)

| Проблема | Эскалировать кому | Канал |
|----------|-------------------|-------|
| KYC / Комплаенс / AML | Susana | Slack ЛС + email |
| Исполнение / техническое MT5 | Pepe | Slack ЛС |
| Депозиты / Выводы / Платежи | Finance Manager | Slack #finance |
| Продажи / Апгрейд / IB | Edward, Franco, Luis | Slack #sales |
| Регуляторная / юридическая жалоба | Susana | Slack ЛС + email (СРОЧНО) |
| Все остальное без решения | Edward | Slack ЛС |

## 6. Решение

### 6.1 Что означает «Решен»

Тикет считается решенным, когда:
- Вопрос клиента получил полный и корректный ответ
- Техническая проблема устранена и клиент подтвердил
- Депозит/вывод обработан корректно
- Жалоба расследована и результат сообщен
- Клиент располагает всей необходимой информацией для продолжения

### 6.2 Информирование о решении

**Чек-лист закрытия:**
- [ ] Подвести итог того, что было решено
- [ ] Подтвердить удовлетворенность клиента
- [ ] Спросить, есть ли дополнительные вопросы
- [ ] Предложить канал для будущих обращений

**Шаблон решения:**
```
Здравствуйте, [имя],

Ваш вопрос решен. [Краткое изложение решения].

[При необходимости: детали предпринятых действий]

Если возникнут дополнительные вопросы или что-то не так, 
не стесняйтесь обращаться. Я к вашим услугам.
```

### 6.3 Тикеты без возможности решения

Если решение неблагоприятно для клиента (необоснованная жалоба, недоступный функционал и т.д.):

```
Здравствуйте, [имя],

После рассмотрения вашего дела, [четкое и честное объяснение ситуации].

Я понимаю, что это не тот ответ, который вы ожидали. [Если есть 
альтернатива: «Что я могу предложить...» / Если нет: «К сожалению, 
в данном случае мы не можем помочь иначе.»]

Если вы не удовлетворены данным ответом, вы можете запросить 
дополнительный пересмотр, и ваше дело будет рассмотрено нашей 
командой [комплаенса/руководства].
```

## 7. Закрытие и сопровождение

### 7.1 Процедура закрытия

| Шаг | Действие | Время |
|-----|----------|-------|
| 1 | Сообщить клиенту о решении | Немедленно |
| 2 | Ожидать подтверждение клиента | 48 часов |
| 3 | При подтверждении: закрыть тикет | Немедленно после подтверждения |
| 4 | При отсутствии ответа: отправить напоминание | Через 48 часов |
| 5 | При отсутствии ответа на напоминание: автоматическое закрытие | 48 часов после напоминания |

**Шаблон напоминания перед закрытием:**
```
Здравствуйте, [имя], хотел убедиться, что ваш вопрос решен. 
Если я не получу ответ в ближайшие 48 часов, тикет будет закрыт. 
Вы можете повторно открыть его в любое время, если потребуется 
дополнительная помощь.
```

### 7.2 Теги закрытия

При закрытии проставить теги результата:

| Тег | Значение |
|-----|----------|
| `resolved-l1` | Решен агентом L1 |
| `resolved-l2` | Решен с участием L2 |
| `resolved-client-confirmed` | Клиент подтвердил удовлетворенность |
| `resolved-no-response` | Клиент не ответил, закрыт по тайм-ауту |
| `unresolved-client-choice` | Клиент решил не продолжать |
| `complaint-resolved` | Жалоба решена |

### 7.3 Сопровождение после закрытия

Для тикетов высокого или срочного приоритета, а также для всех VIP-тикетов:

| Сопровождение | Время | Действие |
|--------------|-------|----------|
| Первое сопровождение | 48ч после закрытия | «Все по-прежнему в порядке с [тема тикета]?» |
| Опрос удовлетворенности | 7 дней после закрытия | Автоматическая отправка из Intercom |

## 8. Опрос удовлетворенности

### 8.1 Конфигурация в Intercom

**Активация:** Intercom > Settings > Messenger > Customer Satisfaction (CSAT)

**Триггер:** Автоматически через 24 часа после закрытия тикета

**Вопрос:** «Как бы вы оценили полученную поддержку?» (5 звезд)

**Открытый вопрос:** «Хотите оставить дополнительный комментарий?»

### 8.2 Использование результатов

| Оценка | Действие |
|--------|----------|
| 5 звезд | Зафиксировать. Рассмотреть запрос публичного отзыва (с разрешения). |
| 4 звезды | Зафиксировать. Специальных действий не требуется. |
| 3 звезды | Edward просматривает тикет для выявления области улучшения. |
| 1-2 звезды | Edward связывается с клиентом для понимания причин неудовлетворенности. Тикет пересматривается внутренне. |

### 8.3 Целевые показатели CSAT

- **Общая цель:** 4.2+ среднемесячный показатель
- **Цель по агенту:** 4.0+ среднемесячный показатель
- **Доля ответов:** 30%+ (нормально, что не все отвечают)

## 9. Управление очередью и приоритизация

### 9.1 Правило приоритизации

Тикеты обрабатываются в следующем порядке:

1. **Срочные** -- Всегда первыми, без исключений
2. **VIP с любым приоритетом** -- Вторыми
3. **Высокий приоритет** -- Третьими
4. **Тикеты с наибольшим временем ожидания** -- Четвертыми (FIFO внутри каждого уровня)
5. **Средний и низкий приоритет** -- Последними

### 9.2 Управление при пиковой нагрузке

Если в очереди больше тикетов, чем агент может обработать:

1. Отправить быстрое подтверждение получения по всем новым тикетам: «Мы получили ваше сообщение. Агент свяжется с вами в ближайшее время.»
2. Приоритизировать согласно правилу приоритизации
3. Уведомить Edward, если более 10 тикетов без ответа
4. В период пересечения смен оба агента совместно обрабатывают очередь

### 9.3 Лимиты одновременных тикетов

| Тип агента | Макс. активных тикетов | Макс. тикетов в ожидании |
|------------|----------------------|--------------------------|
| Агент поддержки | 5 одновременно | 15 в очереди |
| VIP-менеджер | 3 одновременно | 10 в очереди |

При превышении лимита уведомить Edward для перераспределения.

- Обратная связь из опросов удовлетворенности

## 11. Гигиена тикетов

### 11.1 Еженедельная очистка

Каждую пятницу агент должен:

| Действие | Критерий |
|----------|----------|
| Закрыть тикеты в статусе «Ожидание клиента» без ответа >7 дней | Сначала отправить шаблон закрытия |
| Повторно запросить ответ по тикетам в «Ожидание внутреннее» >48ч | Связаться с L2 повторно |
| Проверить назначенные тикеты без активности >72ч | Обновить или закрыть |
| Проверить корректность тегов | Каждый тикет должен иметь тип + приоритет + язык |
| Очистить дублирующиеся тикеты | Объединить, если один клиент создал несколько |

### 11.2 Правила объединения тикетов

Если клиент создал несколько тикетов по одной теме:
1. Определить основной тикет (наиболее полный)
2. Перенести релевантную информацию из дубликата в основной
3. Закрыть дубликат с внутренней заметкой: «Объединен с тикетом #[основной]»
4. Сообщить клиенту: «Я объединил ваши обращения в одно дело #[номер]»
