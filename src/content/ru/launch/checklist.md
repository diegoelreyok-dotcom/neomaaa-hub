# NEOMAAA Broker - Операционный плейбук запуска

## Главный чеклист перед Go-Live

Все, что должно быть готово ДО открытия дверей для реальных клиентов. Упорядочено по приоритету и зависимостям.

---

### БЛОК 1: Фундамент (Недели 1-2)
*Без этого невозможно продвигаться дальше.*

- [ ] **Политика A-Book / B-Book задокументирована** - Pepe пишет внутренний документ. Определяет: какой % остается в B-Book, пороги для перевода в A-Book, как обрабатывается toxic flow, политика хеджирования. Скрипты продаж и compliance зависят от этого.
- [ ] **Схема комиссий утверждена** - Principals утверждают. См. отдельный документ: `NEOMAAA-COMMISSION-SCHEME.md`
- [ ] **PSP активированы и подтверждены** - Финальный список платежных процессоров, которые будут live на день 1. Минимум: 1 crypto (USDT), 1 карта, 1 локальный перевод LATAM (PIX/PSE/SPEI).
- [ ] **Skale CRM настроен для брокера** - Кастомные поля: тип счета, дата FTD, сумма FTD, общие депозиты, последний депозит, статус KYC, назначенный агент, источник лида.
- [ ] **Intercom настроен** - Рабочие процессы поддержки, автоматические ответы, расписание, маршрутизация ES/EN. См.: `NEOMAAA-SUPPORT-PLAYBOOK.md`
- [ ] **Потоки Sumsub финализированы** - KYC-поток для брокера (отличается от prop), принимаемые документы по странам, пороги верификации.

### БЛОК 2: Обучение (Недели 2-6)
*Выполняется параллельно с блоком 1.*

- [ ] **Обучение Sales - Неделя 1/6** завершено (Базовый продукт брокера)
- [ ] **Обучение Sales - Неделя 2/6** завершено (MT5 hands-on)
- [ ] **Обучение Sales - Неделя 3/6** завершено (Compliance + чего НЕ говорить)
- [ ] **Обучение Sales - Неделя 4/6** завершено (Питч + возражения)
- [ ] **Обучение Sales - Неделя 5/6** завершено (CRM + операционный поток)
- [ ] **Обучение Sales - Неделя 6/6** завершено (Симуляции + экзамен)
- [ ] **Обучение Susana** завершено (Compliance брокера vs prop)
- [ ] **Обучение Support Agents** завершено (Intercom + потоки)

### БЛОК 3: Операционные документы (Недели 3-5)
*Можно формировать параллельно с обучением.*

- [ ] **Sales Playbook** финализирован - Скрипты, возражения, питч, KPI. См.: `NEOMAAA-SALES-TRAINING.md`
- [ ] **Compliance Manual** финализирован - KYC/AML, запрещенные фразы, эскалации. См.: `NEOMAAA-COMPLIANCE-WORKFLOW.md`
- [ ] **Support Playbook** финализирован - Типовые ответы, эскалации, SLA. См.: `NEOMAAA-SUPPORT-PLAYBOOK.md`
- [ ] **Client Onboarding Flow** задокументирован - Пошаговый процесс клиента. См.: `NEOMAAA-CLIENT-ONBOARDING.md`
- [ ] **Deposit/Withdrawal Process** задокументирован - См.: `NEOMAAA-DEPOSITS-WITHDRAWALS.md`
- [ ] **Внутренний FAQ** для Support + Sales (50+ часто задаваемых вопросов с утвержденными ответами)

### БЛОК 4: Критичный найм (Недели 1-4)
*Параллельно со всем вышеперечисленным.*

- [ ] **Finance Manager нанят** - ПРИОРИТЕТ #1. Без него брокер работает вслепую.
- [ ] **Support Agent #1 нанят** (LATAM, испанский)
- [ ] **Support Agent #2 нанят** (Europa/Middle East, английский)
- [ ] **Marketing Manager** - решение нанять vs фрилансер принято и реализовано

### БЛОК 5: Тестирование перед запуском (Недели 5-6)
*Ничего не выпускается в продакшн без тестирования.*

- [ ] **Сквозной тест полного потока**: Регистрация > KYC > Депозит > Открытие сделки > Закрытие сделки > Вывод
- [ ] **Тест каждого PSP** с реальной транзакцией (депозит + вывод)
- [ ] **Тест Intercom** - тестовый тикет проходит весь поток эскалации
- [ ] **Тест CRM** - лид поступает, назначается, отслеживается до FTD
- [ ] **Симуляция продажного звонка** - каждый агент проводит 3 оцениваемых mock-звонка
- [ ] **Симуляция KYC** - Susana обрабатывает 10 тестовых KYC с различными сценариями (одобрен, retry, отклонен)
- [ ] **Базовое нагрузочное тестирование** платформы MT5 + client portal

### БЛОК 6: Чеклист Go-Live (День D)

- [ ] Все PSP активны и протестированы
- [ ] MT5 подключен к LP и работает
- [ ] Client portal функционирует (регистрация, KYC, депозиты, выводы)
- [ ] Intercom запущен с залогиненными агентами
- [ ] CRM принимает лиды
- [ ] Susana доступна для одобрения KYC в реальном времени
- [ ] Pepe мониторит dealing desk
- [ ] Franco, Edward, Luis готовы со своими списками контактов на день 1
- [ ] Лендинги с видимыми risk disclaimers
- [ ] Юридические отказы от ответственности в footer всего сайта
- [ ] Метрики дня 1 определены: # регистрации, # завершенных KYC, # FTD, $ депозитов
