## 1. ЗАЧЕМ ИЗУЧАТЬ ЛУЧШИХ

NEOMAAA сегодня работает по лицензии Anjouan, которая имеет менее строгие регуляторные требования по сравнению с CySEC, FCA или ASIC. Тем не менее существуют две веские причины для внедрения compliance на уровне, превышающем минимально необходимый:

1. **Подготовка к расширению:** Если NEOMAAA планирует получить лицензии в Mauritius (FSC) или Seychelles (FSA) и в перспективе CySEC или FCA, необходимо выстраивать масштабируемую инфраструктуру compliance уже сейчас. Ретроспективное внедрение compliance болезненно и дорого.

2. **Репутация и банковские отношения:** PSP и банки-корреспонденты оценивают compliance брокера перед началом сотрудничества. Брокер с надёжным compliance, даже при офшорной лицензии, получает лучшие банковские отношения и более выгодные тарифы.

3. **Защита бизнеса:** Качественная система compliance защищает NEOMAAA от юридических, репутационных и операционных рисков. Один инцидент в сфере compliance способен уничтожить брокера.

## 2. EXNESS -- МУЛЬТИЮРИСДИКЦИОННЫЙ COMPLIANCE В МАСШТАБЕ

### 2.1 Профиль

| Параметр | Детали |
|---|---|
| Основание | 2008 |
| Штаб-квартира | Кипр (CySEC), с глобальными офисами |
| Лицензии | CySEC (Кипр), FCA (Великобритания), FSA (Сейшельские Острова), FSCA (Южная Африка), FSC (Маврикий), FSC (Британские Виргинские Острова), CBCS (Кюрасао) |
| Активные клиенты | 800 000+ (оценка) |
| Ежемесячный объём | $4+ trillion (2024) |
| Команда compliance | 100+ человек (оценка по всем юрисдикциям) |

### 2.2 Структура Compliance

Exness работает по модели compliance по юрисдикциям:

- **Каждая регулируемая структура имеет собственного выделенного Compliance Officer (MLRO).** Compliance не делится между юрисдикциями.
- **Глобальные политики + локальные дополнения:** Имеют глобальную политику AML, которая адаптируется с помощью специфических дополнений по юрисдикциям. Это обеспечивает согласованность без нарушения местного регулирования.
- **Комитет Compliance:** Собирается ежемесячно на глобальном уровне для анализа рисков, инцидентов и регуляторных изменений.
- **Ежеквартальные внутренние аудиты** + ежегодные внешние аудиты по каждой юрисдикции.

### 2.3 KYC и приём на обслуживание

- **Верификация менее чем за 60 секунд** для большинства клиентов (эквивалент Tier 1). Используют агрессивную автоматизацию.
- **Уровни на основе юрисдикции клиента**, а не только суммы депозита. Клиент из ЕС имеет иные требования, нежели клиент из Африки.
- **Liveness check обязателен** для всех уровней.
- **OCR + ИИ** для извлечения данных из документов.
- **Собственная база данных** для обнаружения дубликатов между структурами.

### 2.4 Скрининг

- **Проприетарный скрининг**, дополненный коммерческими базами данных (World-Check, Dow Jones).
- **Скрининг в реальном времени:** Сканируется каждая транзакция, а не только этап приёма на обслуживание.
- **Выделенная команда скрининга:** 15-20 аналитиков, которые занимаются исключительно проверкой совпадений.
- **False positive rate:** <5% после многолетней настройки правил.

## 3. IC MARKETS -- МАССОВЫЙ ОБЪЁМ С НАДЁЖНЫМ COMPLIANCE

### 3.1 Профиль

| Параметр | Детали |
|---|---|
| Основание | 2007 |
| Штаб-квартира | Сидней, Австралия |
| Лицензии | ASIC (Австралия), CySEC (Кипр), FSA (Сейшельские Острова), SCB (Багамские Острова) |
| Активные клиенты | 500 000+ |
| Ежедневный объём | $20+ billion |
| Фокус | Raw spread trading, высокий объём |

### 3.2 Структура Compliance

- **Централизованная модель с региональными кураторами.** Глобальный MLRO с compliance-офицерами по юрисдикциям.
- **Выделенная система compliance:** Не используют таблицы. Имеют внутреннюю систему управления делами (case management) для всех решений compliance.
- **Автоматизированный risk scoring:** Каждый клиент получает оценку риска на основе множества факторов (страна, профессия, депозит, паттерн торговли, возраст, источник средств). Оценка определяет уровень мониторинга.

### 3.3 KYC и приём на обслуживание

- **Уровни, аналогичные NEOMAAA, но с различающимися порогами** в зависимости от юрисдикции.
- **Для ASIC (Австралия):** Полный KYC, включая TFN (Tax File Number) или эквивалент, с первого дня.
- **Для CySEC:** Соответствие 6-й Директиве ЕС по противодействию отмыванию (6AMLD).
- **Для FSA (Сейшельские Острова):** Требования, аналогичные требованиям NEOMAAA в рамках AOFA.
- **Активное использование Refinitiv World-Check** для скрининга санкций и PEP.

### 3.4 Мониторинг транзакций

- **Автоматизированная система мониторинга**, анализирующая:
  - Депозиты и выводы по сумме, периодичности и паттерну.
  - Торговые паттерны (аномальный объём, торговля в нетипичные часы, злоупотребление спредами).
  - Связи между счетами (одинаковый IP, одна и та же карта, один и тот же адрес).
- **Автоматические алерты**, классифицируемые по риску и назначаемые аналитикам.
- **Доля эскалации:** Примерно 2% алертов приводят к формальному расследованию.

## 4. PEPPERSTONE -- СТАНДАРТ FCA/ASIC

### 4.1 Профиль

| Параметр | Детали |
|---|---|
| Основание | 2010 |
| Штаб-квартира | Мельбурн, Австралия |
| Лицензии | FCA (Великобритания), ASIC (Австралия), CySEC (Кипр), BaFin (Германия), DFSA (Дубай), SCB (Багамские Острова), CMA (Кения) |
| Активные клиенты | 400 000+ |
| Фокус | Розничные и профессиональные клиенты, мультирегулирование |

### 4.2 Структура Compliance

- **Структура "трёх линий обороны":**
  - 1-я линия: Front office (продажи, поддержка) -- выявляют риски в своей области.
  - 2-я линия: Команда compliance -- мониторит, проверяет, обеспечивает соблюдение.
  - 3-я линия: Внутренний аудит -- оценивает эффективность первых двух линий.
- **Выделенный MLRO** для каждой регулируемой юрисдикции.
- **Compliance Committee**, отчитывающийся непосредственно перед Board.
- **Культура compliance:** Все сотрудники проходят ежеквартальное обучение AML, не только подразделение compliance.

### 4.3 Дифференцированный KYC по регулятору

| PoA | Счёт за коммунальные услуги или банковская выписка (<3 месяцев) | Счёт за коммунальные услуги или банковская выписка (<3 месяцев) | Аналогично FCA | Meldebescheinigung или аналогичный |
| Оценка пригодности | Обязательна (тест на знания) | Обязательна (тест на знания) | Обязательна | Обязательна + профиль риска |
| Порог SoF | GBP 10,000 | AUD 10,000 | EUR 15,000 | EUR 15,000 |

## 7. СПЕЦИФИЧЕСКИЕ ТРЕБОВАНИЯ AOFA VS ЛУЧШИЕ ПРАКТИКИ

### 7.1 Сравнительная таблица

| Требование | AOFA (минимально необходимое) | Лучшие практики (FCA/ASIC/CySEC) | Рекомендация для NEOMAAA |
|---|---|---|---|
| Письменная политика AML | Да | Да, с ежегодным пересмотром | Внедрить с ежегодным пересмотром |
| KYC при приёме на обслуживание | Да | Да, с уровнями | Внедрить уровни (уже определены) |
| Скрининг санкций | Да | Да, с ongoing monitoring | Внедрить с ongoing monitoring |
| Мониторинг транзакций | Да (базовый) | Да (автоматизированный) | Ручной сейчас, автоматизированный в Фазе 2 |
| SAR регулятору | Да | Да, со строгими сроками | Внедрить формальный процесс |
| Назначенный Compliance Officer | Да | Да, с сертификацией | Susana назначена. Оценить сертификацию. |
| Обучение команды | Требуется (периодичность не определена) | Ежеквартально или ежегодно | Каждые полгода для всей команды |
| Отчёты регулятору | Ежеквартально + ежегодно | Аналогично, более детально | Соблюдать требования AOFA, но с отраслевой детализацией |
| Оценка рисков | Требуется (базовая) | Детальная, по клиенту и по бизнесу | Внедрить оценку по клиенту (ручной risk scoring) |
| Внешний аудит | Ежегодно (финансовая отчётность) | Ежегодно (финансовый + compliance) | Соблюсти минимум, оценить аудит compliance |
| Тест пригодности клиента | Не требуется AOFA [VERIFICAR CON ABOGADO] | Обязателен в FCA/ASIC/CySEC | Не внедрять сейчас, подготовиться к расширению |
| Защита средств клиента | Базовая сегрегация [VERIFICAR CON ABOGADO] | Сегрегированные счета + ICF (Investor Compensation Fund) | Сегрегация средств. Без ICF при лицензии Anjouan. |
| Минимальный капитал | Согласно лицензии [VERIFICAR CON ABOGADO] | EUR 730,000 (CySEC), GBP 730,000 (FCA) | Соблюдать требования AOFA. Планировать капитал для расширения. |

### 7.2 Где NEOMAAA должна превысить минимум AOFA

| Область | Почему превышать минимум | Выгода |
|---|---|---|
| Документирование решений | AOFA не определяет формат, но при аудите это критично | Подготовка к аудитам и расширению |
| Ongoing monitoring санкций | Потому что списки постоянно обновляются | Защита от санкционных рисков |
| Обучение команды | AOFA не определяет периодичность, но продажи взаимодействуют с клиентами | Снижение риска непреднамеренного нарушения |
| Мониторинг транзакций | AOFA требует базовый, но подозрительные паттерны требуют внимания | Защита от отмывания и мошенничества |
| Оценка рисков по клиенту | Позволяет принимать более обоснованные решения | Лучшее управление рисками |
