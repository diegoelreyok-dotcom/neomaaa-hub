### Como se generan los PDFs?

Script: `/Users/gsiglobalteamevent/neomaaa-hub/generate-pdfs.py`
- Python + reportlab
- Procesa todos los .md en `src/content/es/` y `src/content/ru/`
- Output: `public/pdf/es/{section}-{slug}.pdf` y `public/pdf/ru/ru-{section}-{slug}.pdf`
- Excluye: `_sidebar.md`, `README.md`, `404.md`, `dashboard.md`

**No esta integrado al build de Next.js.** Hay que correrlo manual: `python3 generate-pdfs.py`
No hay `npm run pdfs` definido en package.json — solo `build:search` y `build`.

**Regenerar masivo:** ejecutar `python3 generate-pdfs.py` desde la raiz del proyecto. Requiere `pip install reportlab`.

**Gap operativo:** cada vez que se crea/modifica un doc, el PDF queda stale hasta que alguien corra el script. Los 42 docs ES sin PDF son todos docs creados entre el 13-15 abril, despues del ultimo run masivo (commit `0ca7559` del 12 abril "94 PDFs").

## Recomendaciones

### Para PDFs (accion inmediata)
1. **Correr `python3 generate-pdfs.py` ya mismo.** Genera los 42 PDFs ES + 17 RU faltantes. Tarda segundos.
2. **Integrar al build:** agregar a `package.json` un script `"build:pdfs": "python3 generate-pdfs.py"` y correrlo en `postbuild` o como parte de `build`. Problema: requiere Python en el runner de Vercel — no lo tiene por defecto. Alternativa: correrlo en pre-commit hook local o CI dedicado.
3. **Mejor solucion:** portar el generador a Node (puppeteer/playwright con template HTML) y sumarlo al pipeline Next.js. Asi Vercel lo regenera en cada deploy.
